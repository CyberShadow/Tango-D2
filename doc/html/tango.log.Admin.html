@file Admin .d

Classes for administration of logging from a Servlet Engine.

AdminServlet can be used to modify logging details.

AdminContext is a pre-built ServletContext that includes AdminServlet

AdminServer is a pre-built server that allows logging administration. Simply instantiate it with an InternetAddress to bind to.

Scott Sanders, June 3, 2004

  • class AdminServlet : mango.net.servlet.Servlet.Servlet;
    • Servlet to allow modification of Logger details

  • this();
    • Set up the logger for this servlet

  • void addLogger (IServletRequest request, IServletResponse response);
    • Instantiate a new logger, if one does not already exist.

  • void setLoggerLevel (IServletRequest request, IServletResponse response, Level level);
    • Set the level of a given logger, where logger name is in the request parameter 'logger'.

  • void service (IServletRequest request, IServletResponse response);
    • Handle all the different request methods ...

  • class AdminContext : mango.net.servlet.ServletContext.ServletContext;
    • ServletContext prebuilt to include the AdminServlet. Convenience class allowing a developer to drop this prebuilt context into an existing servlet engine. Create a context with a name ("/admin', for example), and AdminContext will provide a "/logger" handler for modifying logging details.

  • class AdminServer ;
    • Given an InternetAddress to bind to, a full admin server is built and deployed. Logging can then be configured at http://address/admin/logger
