tango.util.uuid.NamespaceGenV5

Generate a UUID according to version 5 of RFC 4122.
These UUIDs are generated in a consistent, repeatable fashion. If you generate a version 5 UUID once, it will be the same as the next time you generate it.

To create a version 5 UUID, you need a namespace UUID, generated in some reasonable fashion. This is hashed with a name that you provide to generate the UUID. So while you can easily map names to UUIDs, the reverse mapping will require a lookup of some sort.

This module publicly imports Uuid, so you don't have to import both if you are generating version 5 UUIDs. Also, this module is just provided for convenience -- you can use the method Uuid.byName if you already have an appropriate digest.

Version 5 UUIDs use SHA-1 as the hash function. You may prefer to use version 3 UUIDs instead, which use MD5, if you require compatibility with another application.
